# Phase 0~1 상세 가이드

## Phase 0: 프로토타입 검증

**실행 환경:** Claude 채팅 (claude.ai) — Claude Code가 아님
**목적:** 플레이 가능한 프로토타입으로 게임성을 검증한다.

**프로세스:**
1. 사용자가 레퍼런스, 아이디어를 제시
2. 단일 HTML5 파일로 즉시 구현 (코어 메카닉만, 에셋은 플레이스홀더)
3. 사용자가 플레이하며 수정 요청
4. 코어 메카닉이 재밌다고 확인될 때까지 반복

**산출물:** 검증된 HTML5 프로토타입 (`docs/prototype.html`)

**컨펌 포인트:**
```
"프로토타입의 게임성이 확인되었습니다.
이 HTML을 레퍼런스로 정식 개발을 시작할까요?
'컨펌'이라고 해주시면 Phase 1으로 넘어갑니다."
```

**Phase 1 전환 시:** prototype.html을 `docs/prototype.html`로 복사.
Phase 0은 Claude 채팅에서 진행되므로 Hook 자동 기록 없음. Phase 1 시작 시 work-log에 수동 기록.

## Phase 1: 기획 생성

**산출물:**
- `docs/game-plan.md` — 기획서 (코어루프, 시스템, 경제, 콘텐츠)
- `docs/art-direction.md` — 아트 디렉션 (스타일, 톤, 팔레트)
- `docs/scope-changes.md` — 스코프 변경 추적 (빈 템플릿)

**프로토타입이 있는 경우:** `docs/prototype.html`을 분석하여 기획서에 반영. 확인된 사항은 재질문하지 않는다.

**컨펌 포인트:**
```
"기획서와 아트 디렉션을 작성했습니다. 확인해주세요.
수정할 부분이 있으면 말씀해주시고, 괜찮으면 '컨펌'이라고 해주세요."
```
